# Privacy And Security

Daily Compass is designed for private operating context. Treat real outputs as private unless you intentionally sanitize them.

## Do Not Commit

- raw meeting notes
- transcripts
- private exports
- emails
- credentials
- API tokens
- private filesystem paths
- client names
- internal project names
- proprietary product details
- sensitive screenshots

## Safer Public Examples

Use:

- synthetic names
- mock project labels
- sample meeting notes
- generic roles
- relative sample paths
- short paraphrases

## Source Trails

Private source trails can cite real note links or local paths when appropriate.

Public source trails should cite only mock files, sanitized titles, or generic references.

## Public-Safety Checklist

Before publishing a repo, post, screenshot, or example output, manually check for:

- real company names you did not intend to disclose
- client names
- emails
- private filesystem paths
- raw transcripts
- credentials or tokens
- internal project names
- proprietary product details
- sensitive screenshots
- source links that expose private systems

Automated scans are useful, but not sufficient. Public examples should use mock files only.

## Regulated Or Sensitive Work

For regulated, client-facing, financial, health, legal, or security-sensitive workflows:

- avoid raw transcript ingestion where summaries are enough.
- keep source data local.
- separate public examples from private execution.
- preserve audit trails for real decisions.
- label inferred reminders as inferred.
- do not invent owners, deadlines, or approvals.
