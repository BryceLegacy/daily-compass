# Public Review Checklist

Use this before publishing the repo, screenshots, example outputs, a LinkedIn post, or a longer write-up.

## Product Clarity

- The repo explains Daily Compass in the first screen.
- The quickstart works with sample data only.
- The setup path does not require code or package installation.
- Recurring meetings are user-configurable.
- Stand-up prep is presented as one example, not the whole product.
- Granola is mentioned as the author's tool, not a requirement.
- Local knowledge base usage is optional.

## Privacy

- No real meeting notes.
- No raw transcripts.
- No emails.
- No private filesystem paths.
- No credentials, API tokens, or exported secrets.
- No client names or sensitive partner names.
- No proprietary project or product details.
- No screenshots that reveal private systems.

## Source Trail Safety

- Public examples cite only mock files or sanitized references.
- Private source links are not copied into public docs.
- The prompt tells users to prefer summaries over transcript dumps.
- Inferred reminders are labeled as inferred or uncertain.

## Setup Quality

- A new user can run the sample flow in Codex or Claude.
- A new user can identify whether they need a meeting-note MCP connector.
- A new user can configure recurring meetings.
- Reminder setup is explained without assuming one specific agent.
- The repo makes clear what is required versus optional.

## Launch Readiness

- README is current.
- Example config is current.
- Sample output matches sample inputs.
- Docs do not reference deleted scripts or tools.
- Repo status is clean before publishing.
