# Contributing

Daily Compass is intentionally small and no-code. Contributions should improve the template, docs, sample data, or privacy guidance without adding unnecessary setup burden.

## Good Contributions

- clearer setup instructions
- better sample notes
- safer privacy guidance
- provider-neutral meeting-notes examples
- agent-neutral automation examples
- improvements to the Daily Compass prompt

## Avoid

- real meeting notes
- raw transcripts
- private company or client examples
- credential handling
- dependencies for the basic workflow
- vendor-specific assumptions that make the template less portable

## Before Opening A Pull Request

- Run the sample flow manually with the mock files.
- Check that public examples use only synthetic data.
- Confirm docs do not require code or package installation for the basic workflow.
- Review [Public Review Checklist](docs/public-review-checklist.md).
